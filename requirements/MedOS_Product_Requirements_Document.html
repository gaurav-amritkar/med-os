2.0 API_ROUTER_TOPOLOGY

Express.js routing endpoints. Protected strictly by auth and can(...) RBAC middlewares.

ENDPOINTPAYLOAD EXPECTATIONMIDDLEWARE / ROLESEXECUTION_LOGIC
POST /api/auth/login { username, password } RateLimit(15m/5), None Bcrypt.compare(p, hash) -> Issue 10h JWT payload.
POST /api/encounters/:id/suggest-medicines { disease_description } auth, can('doctor','admin') Trigger Anthropic API. Match against diseaseMedicineMap. Join matching hits with medicine_catalog. Return structured array.
POST /api/pharmacy/dispense { patient_id, medicine_id, qty } auth, can('pharmacist','admin') 1) Select all unexpired batches. 2) Deduct quantity based on FEFO. 3) Write to stock_transactions. 4) Post line-item charge to charges table.
PUT /api/admissions/:id/discharge { discharge_diagnosis } auth, can('doctor','admin') Set status to 'discharged'. Calculate `days = today - admission_date`. Multiply by `room.daily_rate`. Update `room_charges`. Free the room in frontend map.

4.0 GLOBAL_STATE_MANAGEMENT

Zustand acts as the client-side memory volatile store representing the JWT session.

  const useAuthStore = create((set) => ({
    user: null,    // Hydrated from decoded JWT payload
    token: null,   // Active Bearer token
    login: (token, user) => { localStorage.setItem('token', token); set({ token, user }); },
    logout: () => { localStorage.removeItem('token'); set({ token: null, user: null }); },
  }))

5.0 CRITICAL_LOGIC_FUNCTIONS

5.1 PATIENT_LEDGER_SYNC

Financial integrity requires auto-synchronization on any insert/update mutation associated with billing objects.

function syncPatientBalance(patient_id) executes:
  • total_billed = SUM(invoices.total_amount) where patient_id = X
  • total_paid = SUM(payments.amount) where patient_id = X and status = 'success'
  • outstanding = total_billed - total_paid
  • Updates patients record instantly.

5.2 FEFO_INVENTORY_ALGORITHM

Medicine dispensing utilizes a First-Expired-First-Out cascading subtraction matrix.

  const batches = DB.select('id', 'remaining_qty', 'expiry_date').orderBy('expiry_date', 'ASC')
  let required = payload.quantity;
  for (let b of batches) {
    if (required <= 0) break;
    let ded = Math.min(required, b.remaining_qty);
    b.remaining_qty -= ded;
    required -= ded;
    SQL.update({ remaining_qty: b.remaining_qty });
  }
  if (required > 0) throw "INSUFFICIENT_STOCK_EXCEPTION";

7.0 DEPLOYMENT_TOPOLOGY & SECURITY_PARAMETERS

The system is verified to run entirely stateless on the Node.js boundary, leveraging a localized SQLite file backend/medos.db acting as the primary single source of truth. Security mechanisms integrated include:

  • Strict parameterized execution via Knex to nullify all vector SQL injections.
  • Input boundary assertions executing on POST/PUT requests prior to query construction.
  • Express Rate Limiter applied against identity interfaces to mitigate brute forcing.
